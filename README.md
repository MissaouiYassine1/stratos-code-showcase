## 📊 Gestion de Projet Scrum

Ce projet a été géré selon la méthodologie **Scrum**, avec **GitHub Projects** comme outil principal.

### Les 3 Artéfacts Scrum

| Artéfact | Description |
|---|---|
| **Product Backlog** | 13 issues créées (toutes les fonctionnalités) |
| **Sprint Backlog** | 5 tâches sélectionnées par sprint |
| **Increment** | Livraison fonctionnelle à la fin de chaque sprint |

### Les 5 Événements Scrum

| Événement | Fréquence | Objectif |
|---|---|---|
| Sprint Planning | Début de sprint | Choisir les tâches du sprint |
| Daily Scrum | 15 min / jour | "Hier ? Aujourd'hui ? Bloqué ?" |
| Sprint Review | Fin de sprint | Présenter l'incrément |
| Sprint Retrospective | Fin de sprint | Améliorer le processus |
| Sprint | 7 jours | Période de travail focalisée |

### Découpage des Sprints

```
┌─────────────────────────────────────────────────────────────┐
│ SPRINT 1 : Infrastructure (J1–J7)                          │
│ ✅ Installer Next.js | Header | Footer | Responsive        │
├─────────────────────────────────────────────────────────────┤
│ SPRINT 2 : Pages principales (J8–J14)                      │
│ ✅ Services | Contact | À propos                           │
├─────────────────────────────────────────────────────────────┤
│ SPRINT 3 : Optimisation (J15–J21)                          │
│ ✅ Images | Performance SEO | Déploiement Vercel           │
└─────────────────────────────────────────────────────────────┘
```

### GitHub Projects — Tableau Kanban


### Workflow Git / GitHub

```bash
# 1. Créer une branche de fonctionnalité
git checkout -b feature/nom-fonctionnalite

# 2. Commiter avec référence à l'issue
git commit -m "Description, closes #NUMERO"

# 3. Pousser et ouvrir une Pull Request
git push origin feature/nom-fonctionnalite
```

## ⚙️ Installation

### Prérequis

- Node.js 18+
- Git
- Compte GitHub

### Installation locale

```bash
# Cloner le dépôt
git clone https://github.com/votre-pseudo/stratos-code.git

# Accéder au dossier
cd stratos-code

# Installer les dépendances
npm install

# Vérifier les dépendances installées
npm list
```

## 🤝 Contribution

Nous suivons le **GitHub Flow** pour toute collaboration :

1. Forkez le projet
2. Créez votre branche : `git checkout -b feature/AmazingFeature`
3. Commitez vos changements : `git commit -m 'Add AmazingFeature, closes #X'`
4. Poussez la branche : `git push origin feature/AmazingFeature`
5. Ouvrez une **Pull Request**

### Conventions Git

| Type | Format | Exemple |
|---|---|---|
| Feature | `feature/nom-fonctionnalite` | `feature/header-navigation` |
| Bugfix | `bugfix/description` | `bugfix/menu-responsive` |
| Commit | `Action + description + closes #X` | `Ajouter Header responsive, closes #2` |
